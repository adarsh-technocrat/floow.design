# floow.design — AI Mobile App Design Platform

## What This Project Is

floow.design is a SaaS platform where users describe a mobile app in plain English and get pixel-perfect iOS & Android screens generated by AI. Users can iterate via chat, export to Figma, and hand off to developers. **This is specifically a mobile app UI design tool — not a generic web design tool.**

## Tech Stack

- **Framework:** Next.js 16 (App Router, React 19)
- **Language:** TypeScript (strict mode, no `any`)
- **AI:** Google Vertex AI — Gemini 2.5 Pro (chat/thinking) + Gemini 3 Pro Preview (design generation)
- **Database:** PostgreSQL on Neon, via Prisma ORM
- **Auth:** Firebase (Google, GitHub, Apple, Email)
- **Payments:** Stripe (subscriptions, webhooks, customer portal)
- **Storage:** Vercel Blob (images, thumbnails, blog assets)
- **Email:** Resend (transactional emails)
- **State:** Redux Toolkit (7 slices)
- **Styling:** Tailwind CSS 4 (inline `@theme` in globals.css, CSS variables for light/dark)
- **Package Manager:** pnpm

## Environment Variables

All env vars are in `.env.local` (see `.env.example` for reference). Key groups:

- `DATABASE_URL` — Neon PostgreSQL
- `NEXT_PUBLIC_FIREBASE_*` — Firebase Auth (7 vars)
- `GOOGLE_CLIENT_EMAIL`, `GOOGLE_PRIVATE_KEY`, `GOOGLE_VERTEX_PROJECT` — Vertex AI
- `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`, `STRIPE_PRICE_*` — Stripe
- `BLOB_READ_WRITE_TOKEN` — Vercel Blob
- `RESEND_API_KEY` — Email

Scripts that need env vars use: `npx dotenv -e .env.local -- npx tsx scripts/<file>.ts`

## Database

PostgreSQL via Prisma. Key models:

- **User** — auth, plan (FREE/LITE/STARTER/PRO/TEAM), credits, Stripe IDs
- **Project** — design projects with frames, can be templates
- **Frame** — individual screens (HTML stored as text), positioned on canvas
- **ChatSession** — conversation history per project/user (JSON messages)
- **Team** — multi-user workspaces with roles (OWNER/ADMIN/MEMBER)
- **BlogPost** — published articles (slug, content, tags, coverImage, published flag)
- **Theme** — saved design system variables (JSON)

After schema changes: `pnpm db:push` (dev) or `pnpm db:migrate` (production)

## AI Design System

The core AI pipeline lives in `src/lib/agent/`:

- **tools.ts** (42KB) — Tool definitions: create_all_screens, create_screen, update_screen, generate_image, save_theme, inspect_element, etc.
- **planner.ts** — Multi-step design breakdown for complex prompts
- **image-gen.ts** — Google Imagen for image generation, uploads to Vercel Blob
- System prompts in `src/constants/agent-prompts.ts`

The chat endpoint (`POST /api/chat`) streams Gemini responses with extended thinking (8192 tokens).

## Blog System

Blog posts are stored in PostgreSQL (BlogPost model) and rendered via MDX (`next-mdx-remote` with `remark-gfm`). The blog page (`/blog`) fetches all published posts via `getAllPosts()` — no static file indexing needed.

**Seeding posts:** Create a script in `scripts/seed-<name>.ts` using Prisma's `upsert`, run with:

```bash
npx dotenv -e .env.local -- npx tsx scripts/seed-<name>.ts
```

**Uploading blog images:** Use `@vercel/blob` `put()` with `addRandomSuffix: false, allowOverwrite: true`, then update the post's `coverImage` field.

**Blog publish pipeline:** `pnpm blog:publish` uses Gemini 2.5 Flash with Google Search grounding to auto-generate articles.

## Content & SEO Positioning

**floow.design is an AI mobile app UI design tool.** All blog content, CTAs, and marketing copy must position it specifically for mobile app design — not generic web/UI design.

Standard CTA angle for blog posts:

> "Design your mobile app screens in floow.design first, then build/export."

Competitor blogs use the competitor's slug (e.g., `/blog/magic-patterns-alternatives` matches banani.co's URL). Blog posts include Floow mentions in "gap" sections where competitors fall short on mobile UI.

## Deployment

Hosted on Vercel. Pushes to `main` trigger automatic deployment. No `vercel.json` — uses default Next.js config. Images served via Vercel Blob CDN.

## Important Patterns

- **Protected routes** use `(app)` route group with AuthGuard
- **Public routes** use `(landing)` route group
- **API auth** syncs Firebase tokens to Prisma User via `POST /api/auth/sync`
- **Credits** are tracked per-user with daily reset, logged in CreditLog
- **Teams** share credits and projects; invites are token-based with expiry
- **Frames** store raw HTML as text — rendered in iframes on the canvas
- **Blog images** go to Vercel Blob, not `public/` — use blob URLs in `coverImage`
